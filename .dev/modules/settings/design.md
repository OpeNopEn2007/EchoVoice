# Settings 模块设计文档

## 模块职责

提供用户配置界面，是 EchoVoice 桌面端的设置面板。允许用户自定义热键、提示音、模型等选项。

## 设计方案

### 实现方式

采用 **Tauri Webview** 实现设置面板：
- 优点：与 Tauri 集成好，轻量，支持跨平台
- 使用 HTML/CSS/JS 开发简单界面
- 通过 Tauri 命令与 Rust 后端通信

### 窗口属性

| 属性 | 值 |
|------|-----|
| 标题 | EchoVoice 设置 |
| 默认尺寸 | 600 x 500 |
| 最小尺寸 | 400 x 300 |
| 可调整大小 | 是 |
| 居中显示 | 是 |
| 模态窗口 | 否 |

## 技术实现

### 前端技术

- **框架**: 原生 HTML + CSS（不使用框架，保持轻量）
- **样式**: CSS 变量，支持主题切换

### 前后端通信

```rust
// Tauri 命令
#[tauri::command]
fn get_config() -> Config;

#[tauri::command]
fn save_config(config: Config) -> Result<(), String>;

#[tauri::command]
fn register_hotkey(key: String) -> Result<(), String>;

#[tauri::command]
fn play_sound(sound_type: String) -> Result<(), String>;
```

### 事件流

1. 用户打开设置面板 → 前端请求当前配置
2. Rust 后端读取配置文件 → 返回 JSON
3. 前端渲染设置界面
4. 用户修改设置 → 前端发送保存请求
5. Rust 后端验证并保存配置 → 通知相关模块重新加载

---

## 错误处理

| 场景 | 处理方式 |
|------|----------|
| 配置文件损坏 | 使用默认配置，显示警告 |
| 快捷键冲突 | 提示用户并阻止保存 |
| 权限不足 | 提示用户授权 |
| 保存失败 | 显示错误信息，保留原值 |
