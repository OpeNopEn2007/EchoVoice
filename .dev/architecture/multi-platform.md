# 多端策略

> EchoVoice 桌面端与移动端的技术差异设计

---

## 开发顺序

### 阶段一：桌面端开发（当前）✅ 优先

1. **Windows + macOS 双平台**
   - 首先实现跨平台基础功能
   - 验证核心流程可用

2. **必须包含设置面板**
   - 热键配置
   - 提示音开关
   - 模型选择
   - 主题/透明度
   - 开机自启动

### 阶段二：移动端开发（待开发）

1. **iOS + Android 双平台**
   - 基于 Flutter 实现
   - 悬浮球触发机制

---

## 设计目标

| 指标 | 桌面端 | 移动端 |
|------|--------|--------|
| 目标用户 | PC 用户 | 手机用户 |
| 运行方式 | 后台守护 | 前台服务 |
| UI 交互 | 托盘 + 快捷键 | 悬浮球 |

---

## 桌面端设计

### 技术栈

- **框架**: Tauri + Rust
- **模型**: Whisper Base + SmolLM2-1.7B
- **内存占用**: ~2.5GB

### 交互方式

1. **启动**: 开机自启动，后台守护
2. **触发**: 全局快捷键（F9 / Capslock）
3. **录音**: 按住说话，释放停止
4. **输出**: 模拟键盘输入 / 复制到剪贴板

### 特性

- 完整的系统托盘菜单
- 设置面板（快捷键自定义）
- 最小化到托盘
- Windows / macOS / Linux 全平台支持

---

## 移动端设计

### 技术栈

- **框架**: Flutter
- **模型**: Whisper Tiny + SmolLM2-360M
- **内存占用**: ~600MB

### 交互方式

1. **启动**: APP 常驻后台
2. **触发**: 悬浮球长按
3. **录音**: 按住说话，释放停止
4. **输出**: 直接上屏

### 特性

- 智能悬浮球（不遮挡内容）
- 透明度可调
- iOS / Android 双平台

---

## 模型差异

| 模型 | 桌面端 | 移动端 |
|------|--------|--------|
| Whisper | Base (74MB) | Tiny (31MB) |
| SmolLM2 | 1.7B (1GB) | 360M (250MB) |
| 识别质量 | 高 | 中等 |
| 润色质量 | 高 | 中等 |
| 推理速度 | 快 | 优化后可用 |

## 扩展性设计

### 模型热插拔

```rust
// 配置文件中指定模型路径
asr:
  model: "models/whisper-base.bin"

llm:
  model: "models/smollm2-1.7b.gguf"
```

## 代码复用策略

```
crates/
├── audio/        # 共用
├── asr/          # 共用（不同模型）
├── llm/          # 共用（不同模型）
├── hotkey/       # 桌面专用
├── tray/         # 桌面专用
└── floating/     # 移动端专用
```

移动端和桌面端共享核心业务逻辑（audio、asr、llm），仅 UI 层和触发机制不同。
