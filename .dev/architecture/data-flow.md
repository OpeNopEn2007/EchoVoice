# 数据流设计

> 从录音到上屏的完整流程

---

## 主流程

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  触发   │ →  │  录音   │ →  │  ASR    │ →  │  LLM    │ →  │  上屏   │
│ 快捷键  │    │  采集   │    │  识别   │    │  润色   │    │  输出   │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
     │              │              │              │              │
     ▼              ▼              ▼              ▼              ▼
  状态检查      音频数据       原始文本       润色文本       键盘事件
```

---

## 详细步骤

### Step 1: 触发检测

**输入**: 用户按下快捷键（F9 或 Capslock）
**处理**:
1. 检查当前是否正在录音
2. 如果未录音 → 进入 Step 2（开始录音）
3. 如果正在录音 → 进入 Step 3（停止录音并处理）

**状态管理**:
```rust
enum State {
    Idle,       // 空闲
    Recording,  // 录音中
    Processing, // 处理中（ASR + LLM）
}
```

---

### Step 2: 开始录音

**输入**: 触发信号
**处理**:
1. 初始化音频录制器（cpal）
2. 打开默认麦克风
3. 开始录制音频流（f32格式，16kHz）
4. 音频数据存入缓冲区

**输出**: 音频流持续写入缓冲区

**错误处理**:
- 无麦克风 → 提示用户
- 权限拒绝 → 引导用户开启权限
- 设备占用 → 等待或提示

---

### Step 3: 停止录音

**输入**: 用户释放快捷键
**处理**:
1. 停止音频流
2. 获取缓冲区中的音频数据（Vec<f32>）
3. 释放音频设备

**输出**: 完整音频数据（Vec<f32>）

---

### Step 4: ASR 识别

**输入**: 音频数据（Vec<f32>）
**处理**:
1. 检查 Whisper 模型是否已加载
2. 如果未加载 → 懒加载模型（Whisper tiny）
3. 调用 whisper.cpp 进行识别
4. 返回识别文本

**输出**: 原始文本（String）

**性能要求**:
- 识别时间 < 音频长度的 0.5 倍（实时因子 0.5x）
- 准确率 > 90%（中文）

---

### Step 5: LLM 润色

**输入**: 原始文本（String）
**处理**:
1. 检查 SmolLM2 模型是否已加载
2. 如果未加载 → 懒加载模型
3. 构造提示词（系统提示 + 用户输入）
4. 调用 llama.cpp 进行推理
5. 提取润色后的文本

**提示词模板**:
```
你是一个智能文本助手。请对用户输入进行润色：
1. 将口语化的标点词转换为实际标点（如"逗号"→"，"）
2. 适当分段，使文本更易读
3. 保持原意，不做过度修改

用户输入: {text}
润色结果:
```

**输出**: 润色文本（String）

**性能要求**:
- 推理时间 < 2秒（短文本）
- 不改变原意

---

### Step 6: 键盘上屏

**输入**: 润色文本（String）
**处理**:
1. 使用 enigo 模拟键盘输入
2. 逐字符或整段输入文本
3. 文本出现在当前光标位置

**输出**: 文本上屏完成

---

## 异常流程

### 异常 1: 录音失败

```
开始录音 → 设备不可用 → 提示用户 → 返回空闲状态
```

### 异常 2: ASR 识别失败

```
停止录音 → ASR失败 → 使用原始音频（不上屏）→ 记录日志 → 提示用户
```

### 异常 3: LLM 润色失败

```
ASR成功 → LLM失败 → 使用ASR原始文本上屏 → 记录日志
```

### 异常 4: 上屏失败

```
LLM成功 → 上屏失败 → 复制到剪贴板 → 提示用户手动粘贴
```

## 数据格式

### 音频数据
- 格式: f32 数组（-1.0 到 1.0）
- 采样率: 16000 Hz（Whisper要求）
- 通道: 单声道

### 文本数据
- 编码: UTF-8
- 语言: 中文/英文混合
