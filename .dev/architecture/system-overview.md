# 系统概览

> EchoVoice 整体架构一句话描述

---

## 一句话定义

EchoVoice 是一个**纯本地运行的轻量级 AI 语音输入法**，通过快捷键触发录音，经 ASR 识别和 LLM 润色后，自动模拟键盘输入上屏。

---

## 核心特性

| 特性 | 说明 |
|------|------|
| **纯本地** | 所有模型本地运行，无需联网 |
| **轻量级** | 手机端 <600MB 内存，桌面端 <2.5GB |
| **跨平台** | iOS / Android / Windows / macOS / Linux |
| **隐私优先** | 语音数据永不离开设备 |
| **无缝集成** | 全局快捷键，不干扰原有输入法 |

## 技术栈

| 层级 | 技术 | 选型理由 |
|------|------|---------|
| 桌面端 | Tauri + Rust | 轻量、安全、跨平台 |
| 移动端 | Flutter | 一套代码双端、性能好 |
| ASR | Whisper.cpp | 本地运行、多语言、小巧 |
| LLM | llama.cpp + SmolLM2 | 边缘优化、指令遵循强 |

---

## 数据流概要

1. **触发**: 用户按下快捷键（F9/Capslock）或长按悬浮球
2. **录音**: 开始录制麦克风音频
3. **停止**: 用户释放快捷键，停止录音
4. **识别**: ASR 将音频转为文本
5. **润色**: LLM 结构化整理文本（标点、分段）
6. **上屏**: 模拟键盘输入，文本出现在当前光标位置

详细流程参见 [数据流设计](./data-flow.md)

## 多端差异

| 特性 | 桌面端 | 移动端 |
|------|--------|--------|
| 触发方式 | 全局快捷键 | 悬浮球长按 |
| 后台运行 | 是 | 是（前台服务）|
| 模型大小 | 1.7B (完整) | 360M (轻量) |
| 内存占用 | ~2.5GB | ~600MB |
| UI界面 | 无（纯后台）| 悬浮球 |

详细策略参见 [多端策略](./multi-platform.md)
