## 检查清单

每次 `/loop` 执行时检查：

1. **当前阶段是否完成？**
   - 是 → 进入下一阶段
   - 否 → 继续当前阶段

2. **是否有编译错误？**
   - 是 → 修复错误
   - 否 → 继续开发

3. **是否有功能异常？**
   - 是 → 调试修复
   - 否 → 继续开发

4. **是否需要用户确认？**
   - 是 → 等待用户输入
   - 否 → 自动继续

---

## 问题处理策略

### 编译错误
- 尝试自动修复
- 如果无法自动修复 → 记录错误 → 等待用户

### 设计决策
- 如果有明确方案 → 直接实现
- 如果有多种方案 → 简要说明利弊 → 等待用户选择

### 依赖缺失
- 尝试自动安装（cargo）
- 如果系统级依赖缺失 → 告知用户手动安装

### 测试失败
- 自动调试修复
- 如果无法修复 → 记录问题 → 继续其他任务
